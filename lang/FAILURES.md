# CSS Grammar Failure Analysis

Summary of the 603 parse failures found by running `go test ./lang/...` against the
`lang/testdata` corpus. Each category includes a count, a brief description, a minimal
example of a failing snippet, and the fix required.

Generated by `scripts/analyze_failures.py` against `temp/out.txt`.

---

## Documented constraints (121 failures)

These are inherent EBNF representation limitations already documented in
`docs/CSS_EBNF_CONSTRAINTS.md`. They are not bugs; fixing them would require
either restructuring large parts of the grammar or moving the constraint to the
semantic/render layer.

### `ebnf_constraint__vds_ordering` — 97 failures

VDS `||` / `&&` operators allow any order; the EBNF translation fixes a canonical
order. CSS that uses a non-canonical value order fails.

```css
/* animation shorthand: spec allows value-order freedom; EBNF fixes one order */
div { animation: slide 1s ease-in-out 0s infinite; }
```

*Documented in `CSS_EBNF_CONSTRAINTS.md` §"|| and && both lose ordering flexibility".*

### `ebnf_constraint__vendor_prefix_property` — 24 failures

Vendor-prefixed property declarations (`-webkit-*`, `-moz-*`, `-ms-*`) are not
written into the grammar. Their documentation is inconsistent and they duplicate
standard rules.

```css
div { -webkit-border-before: 1px solid black; }
```

*Documented in `CSS_EBNF_CONSTRAINTS.md` §"WebKit extensions are not written".*

### `ebnf_constraint__url_prefix_ambiguity` — 1 failure

The optional `[ NamespacePrefixType ]` in `@namespace` consumes `url` as a valid
identifier before the `url(...)` form can be tried.

```css
@namespace url("http://www.w3.org/1999/xhtml");
```

*Documented in `CSS_EBNF_CONSTRAINTS.md` §"Optional rules greedily consume shared identifier prefixes".*

## Undocumented constraints (126 failures)

Real EBNF/parser limitations that are not yet in `CSS_EBNF_CONSTRAINTS.md`.

### `declarationvaluetype_placeholder` — 95 failures

`DeclarationValueType` in `functions.ebnf` is defined as `string_literal`, so it
only matches quoted strings. At-rules that use it as a free-form value bucket
(`@media`, `@supports`, `@container`, `@custom-media`, `@property`, etc.) fail on
any unquoted value.

```css
/* @property descriptor values are not quoted strings */
@property --myColor {
  syntax: "<color>";
  inherits: true;
  initial-value: rebeccapurple;
}
```

**Fix:** Replace `DeclarationValueType = string_literal` with a rule that accepts
any CSS token sequence (idents, numbers, symbols, strings, functions, …).

### `keyframes_percent_selector` — 31 failures

`AtKeyframes` routes its block through `QualifiedRuleListType = CssStyleSheet`.
`CssStyleSheet` uses `ComplexSelectorList` as the rule head, which cannot match
percentage keyframe selectors (`0%`, `50%`, `100%`, `from`, `to`).

```css
@keyframes slide {
  0%   { transform: translateX(0); }
  100% { transform: translateX(200px); }
}
```

**Fix:** Add a dedicated `KeyframeBlock` rule with `KeyframeSelector = percentage_type | from | to`.

### `viewport_unit_keyword_boundary` — 2 failures

`sv`, `lv`, `dv` are defined as keyword terminals in `keyword.ebnf`. Gluon
enforces a word boundary after each keyword match, so `svh`, `dvw`, `lvmin`, etc.
fail because the parser matches `sv`/`dv`/`lv` and then rejects the continuation.

```css
body { min-height: 100svh; }
```

**Fix:** Remove `sv`, `lv`, `dv` as standalone keywords and define
`small_viewport_length_unit`, `large_viewport_length_unit`,
`dynamic_viewport_length_unit` as full multi-character lexical rules instead.

## Naming bugs (101 failures)

Errors in the EBNF rules themselves — correct concept, wrong name or wrong rule
referenced.

### `naming_bug__image_type_uses_url_keyword` — 88 failures

`ImageType` in `datatype.ebnf` references the bare keyword terminal `url`
(`"url"`) instead of the syntactic rule `UrlType` (`url , "(" , string_type , ")"`).
Every `background-image: url("…")`, `border-image: url("…")`, `content: url("…")`,
etc. fails immediately after the `url` token because nothing consumes the `(`.

```css
div { background-image: url("bg.png"); }
```

**Fix:** `ImageType = UrlType | ImageFn | CrossFadeFn | ...`

### `naming_bug__trig_fn_not_in_calc_value` — 7 failures

`CosFn`, `SinFn`, `TanFn`, `AcosFn`, `AsinFn`, `AtanFn`, `Atan2Fn`, `ExpFn`,
`LogFn`, `SqrtFn` are defined in `functions.ebnf` but are not listed in
`CalcValueType` or `MathFunctionType` in `datatype.ebnf`.

```css
div {
  width: calc(cos(45deg) * 100px);
  transform: rotate(atan2(1, 1));
}
```

**Fix:** Add all trig/exponential functions to `MathFunctionType` and ensure
`CalcValueType` references `MathFunctionType`.

### `naming_bug__font_face_stretch_vs_width` — 4 failures

`AtFontFaceWidthDesc` uses the keyword `font_width = "font-width"` but real CSS
uses `font-stretch` in `@font-face` blocks. The descriptor rule never matches.

```css
@font-face {
  font-family: "MyFont";
  src: url("font.woff2");
  font-stretch: 75% 125%;  /* fails — grammar looks for font-width */
}
```

**Fix:** Rename `AtFontFaceWidthDesc` to use `font_stretch` and update the keyword.

### `naming_bug__srgb_case` — 2 failures

`sRGB = "sRGB"` in `keyword.ebnf` is case-sensitive. CSS `color(srgb …)` uses all
lowercase, so it never matches.

```css
div { background-color: color(srgb 1 0 0); }
```

**Fix:** Change to `srgb_kw = "srgb"` (or make the terminal case-insensitive).

## Missing rules (169 failures)

Valid CSS constructs that have no grammar rule at all.

### `unicode_in_string_literal` — 29 failures

`string_char_dq` and `string_char_sq` in `primitive.ebnf` are defined as
`" " ... "~"` (ASCII 0x20–0x7E). Unicode characters (e.g. circled letters in
`@counter-style symbols`, accented characters in font-family strings) are above
0x7E and are rejected.

```css
@counter-style circled-alpha {
  system: fixed;
  symbols: Ⓐ Ⓑ Ⓒ Ⓓ Ⓔ;  /* U+24B6… rejected by string_char rule */
}
```

**Fix:** Extend string character ranges to cover the full Unicode range above 0x7E,
e.g. add `| "\u0080" ... "\uFFFF"` (and surrogate pairs for SMP if needed).

### `missing_at_rule_descriptor__counter_style` — 16 failures

`@counter-style` blocks use a dedicated `AtCounterStyleBlock` but it is missing the
`additive-symbols`, `fallback`, `suffix`, and `speak-as` descriptors.

```css
@counter-style lower-roman {
  system: additive;
  additive-symbols: 1000 M, 900 CM;  /* no rule for additive-symbols */
  suffix: ". ";
}
```

**Fix:** Add `AtCounterStyleAdditiveSymbolsDesc`, `AtCounterStyleFallbackDesc`,
`AtCounterStyleSuffixDesc` to `AtCounterStyleBlock`.

### `shape_function` — 11 failures

Shape functions (`polygon()`, `circle()`, `ellipse()`, `inset()`, `path()`,
`shape()`) appear as `clip-path` and `offset-path` values but are not in
`ClipPathProp` or the relevant property rules. A `ShapeFunctionType` exists in
`datatype.ebnf` but is not wired in.

```css
div { clip-path: polygon(50% 0%, 100% 100%, 0% 100%); }
```

**Fix:** Add `ShapeFunctionType` as an alternative in `ClipPathProp`,
`OffsetPathProp`, and any other property that accepts `<basic-shape>`.

### `missing_at_rule_descriptor__font_face` — 9 failures

`AtFontFaceBlock` is missing `font-display`, `font-variation-settings`,
`size-adjust`, and `unicode-range` descriptors.

```css
@font-face {
  font-family: "web-font";
  src: url("font.woff2");
  font-display: swap;          /* no rule */
  unicode-range: U+0025-00FF;  /* no rule */
}
```

**Fix:** Add descriptor rules for `font-display`, `font-variation-settings`,
`size-adjust`, and `unicode-range` to `AtFontFaceBlock`.

### `missing_at_rule_descriptor__page_size` — 8 failures

`AtPage` handles margins but has no rule for the `size` or `page-orientation`
descriptors.

```css
@page { size: landscape; margin: 2cm; }
```

**Fix:** Add `AtPageSizeDesc = size , colon_symbol , PageSizeType , semicolon_symbol`
and include it in the `@page` block.

### `missing_at_rule_descriptor__font_palette_values` — 7 failures

`@font-palette-values` has no rules for the `base-palette` or `override-colors`
descriptors.

```css
@font-palette-values --Alternate {
  font-family: "Bungee Spice";
  base-palette: 1;
  override-colors: 0 #ff0, 1 #f00;
}
```

**Fix:** Add a dedicated `AtFontPaletteValuesBlock` with descriptor rules.

### `gradient_multiposition_stop` — 7 failures

CSS Images Level 4 allows a color stop to carry two positions
(`red 20% 40%`). The `ColorStopType` rule only accepts one position.

```css
div {
  background: linear-gradient(red 20% 40%, blue);
}
```

**Fix:** Update `ColorStopType` to accept an optional second position:
`color_value , position_type , [ position_type ]`.

### `missing_math_function` — 5 failures

`abs()` and `sign()` are in CSS Values Level 4 but have no rule in `functions.ebnf`.
(Trig functions are defined but not wired in — see naming bug above.)

```css
div { background-image: linear-gradient(abs(var(--deg)), blue, red); }
```

**Fix:** Add `AbsFn` and `SignFn` rules and include them in `MathFunctionType`.

### `missing_at_rule_descriptor__property` — 5 failures

`AtProperty` has no block rule for its required descriptors (`syntax`, `inherits`,
`initial-value`).

```css
@property --progress {
  syntax: "<percentage>";
  inherits: false;
  initial-value: 25%;
}
```

**Fix:** Add `AtPropertyBlock` with `syntax`, `inherits`, and `initial-value`
descriptor rules.

### `color_interpolation_method` — 5 failures

`color-mix()` and gradient `in <colorspace>` syntax accept a
`<color-interpolation-method>` (`in srgb`, `in oklch longer hue`, etc.) but the
grammar only accepts bare colorspace identifiers, not the `in` keyword or hue
interpolation modifiers.

```css
.srgb { background-image: linear-gradient(in srgb to right, blue, red); }
.oklch { background-image: linear-gradient(in oklch longer hue, blue, red); }
```

**Fix:** Add a `ColorInterpolationMethodType` rule covering `in <colorspace>
[ <hue-interpolation-method> ]` and use it in `GradientFnType` and `ColorMixFn`.

### `dashed_function_call` — 5 failures

Custom CSS functions (`--fn-name()`) introduced in CSS Values Level 5 have no rule
in the grammar. They appear in `@custom-media` values and inside `calc()`.

```css
@custom-media --narrow-window (width < 32em);
@media (--narrow-window) { }
```

**Fix:** Add `CustomFunctionCall = dashed_ident_type , left_parenthesis_symbol ,
{ CustomPropertyValue } , right_parenthesis_symbol` and include it in value
positions where functions are accepted.

### `is_with_pseudo_element` — 4 failures

`:is()` and `:where()` only accept a `ForgivingSelectorListType` which is built
from `ComplexSelector`. Pseudo-elements (`::before`, `::after`) inside `:is()` are
rejected because `ComplexSelector` does not include pseudo-element tails.

```css
:is(h1, h2, h3)::before { content: "»"; }
```

**Fix:** Allow pseudo-element tails in the forgiving selector list used by `:is()`
and `:where()`.

### `missing_at_rule_descriptor__function_result` — 3 failures

`@function` (CSS Functions Level 1) uses a `result:` pseudo-descriptor to specify
the return value. No rule handles this.

```css
@function --circle-area(--r) {
  result: calc(pi * var(--r) * var(--r));
}
```

**Fix:** Add a `result` keyword and a `AtFunctionResultDesc` rule to the `@function`
block.

### `nth_of_selector` — 3 failures

`:nth-child(An+B of <selector-list>)` (CSS Selectors Level 4) is not handled; only
the bare `An+B` form is parsed.

```css
li:nth-child(2 of .important) { color: red; }
```

**Fix:** Add the optional `of ForgivingSelectorListType` tail to `PcNthChild` and
`PcNthLastChild`.

### `system_color` — 3 failures

CSS system color keywords (`Canvas`, `ButtonFace`, `CanvasText`, etc.) are not
included in `ColorType`.

```css
button { background-color: ButtonFace; color: ButtonText; }
```

**Fix:** Add a `SystemColorType` rule and include it as an alternative in
`ColorType`.

### `vendor_pseudo_class` — 3 failures

Vendor-prefixed single-colon pseudo-classes (`:-webkit-*`, `:-moz-*`) start with
`:-` but no rule handles them.

```css
img:-moz-loading { opacity: 0; }
```

**Fix:** Add a `PcVendor = colon_symbol , hyphen_symbol , ident_type` rule (mirroring
the existing `PeVendor`) as a fallback in `PseudoClass`.

### `missing_pseudo_element_rule` — 3 failures

`::search-text`, complex `::cue(…)` with arguments, and `::part(…)` with multiple
identifiers are not defined in `pseudo-element.ebnf`.

```css
::search-text { background-color: yellow; }
::cue(b) { color: red; }
::part(base inner) { color: blue; }
```

**Fix:** Add `PeSearchText`, update `PeCue` to accept an optional selector argument,
and update `PePart` to accept multiple ident arguments.

### `missing_at_rule__color_profile_in_color_fn` — 2 failures

`color()` only accepts a fixed set of predefined color spaces. Custom color spaces
defined via `@color-profile` (e.g. `--swop5c`) cannot be used as the color-space
argument.

```css
@color-profile --swop5c { src: url("https://example.org/SWOP2006_Coated5v2.icc"); }
div { color: color(--swop5c 0.1 0.5 0.2 0.4); }
```

**Fix:** Allow `dashed_ident_type` as a color-space alternative in `ColorFn`.

### `missing_at_rule_descriptor__font_feature_values` — 2 failures

`@font-feature-values` contains nested at-rules (`@swash`, `@ornaments`,
`@annotation`, etc.) that the grammar does not support.

```css
@font-feature-values Font One {
  @styleset { nice-style: 12; }
}
```

**Fix:** Add a dedicated `AtFontFeatureValuesBlock` rule with nested at-rule
handling.

### `missing_at_rule_descriptor__view_transition` — 2 failures

`@view-transition` is defined but only supports the `navigation` descriptor, not
the `types` descriptor.

```css
@view-transition { navigation: auto; types: slide, forwards; }
```

**Fix:** Add a `types` descriptor rule to `AtViewTransitionBlock`.

### `calc_size_function` — 2 failures

`calc-size()` (CSS Values Level 5) is not in the grammar.

```css
section { width: calc-size(fit-content, size + 1rem); }
```

**Fix:** Add a `CalcSizeFn` rule and include it wherever `calc()` is accepted.

### `lang_pseudo_extended_syntax` — 2 failures

`:lang()` only accepts a single ident argument. CSS Selectors Level 4 allows a
comma-separated list, string arguments, and wildcard (`*`).

```css
:lang("zh-Hans", "zh-Hant") { font-family: "Noto Sans CJK"; }
```

**Fix:** Update `PcLang` to accept `( ident_type | string_type | asterisk_symbol ) ,
{ comma_symbol , ( ident_type | string_type | asterisk_symbol ) }`.

### `escaped_identifier` — 2 failures

Escaped characters in identifiers (e.g. `\?`, `\000031`) are not handled by
`ident_type` in `primitive.ebnf`.

```css
.item\?one { background-color: pink; }
```

**Fix:** Extend `ident_start` / `ident_continue` to allow `\` followed by a hex
sequence or escaped character.

### `vendor_function` — 6 failures

Vendor-prefixed CSS functions (`-moz-image-rect()`, `-webkit-gradient()`, etc.)
have no grammar rules.

```css
div { background-image: -moz-image-rect(url("sprite.png"), 0, 16, 16, 0); }
```

**Fix:** Add a catch-all `VendorFunctionCall = hyphen_symbol , ident_type ,
left_parenthesis_symbol , { any_value } , right_parenthesis_symbol` or enumerate the
most common ones individually.

### `missing_at_rule__document` — 1 failure

`@-moz-document` is non-standard (dropped from spec). It appears in real-world CSS.
Not worth adding to the grammar; mark as acceptable failure.

### `missing_at_rule__css_nesting` — 1 failure

`@starting-style`, `@layer`, and `@supports` inside a rule body (CSS nesting) are
not handled in `RuleBody`.

```css
.card { @starting-style { opacity: 0; } }
```

**Fix:** Add `AtLayer`, `AtSupports`, and `AtStartingStyle` as alternatives in
`RuleBody` for nested use.

### `color_mix_var_in_method` — 1 failure

`color-mix()` does not allow `var()` inside the color-interpolation-method argument.

```css
div { color: color-mix(in var(--space), red, blue); }
```

**Fix:** Allow `VarFn` as an alternative in the color-space position of `ColorMixFn`.
