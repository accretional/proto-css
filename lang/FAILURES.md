# CSS Grammar Failure Analysis

Summary of the 603 parse failures found by running `go test ./lang/...` against the
`lang/testdata` corpus. Generated by `scripts/analyze_failures.py` against `temp/out.txt`.

---

## Documented constraints (122 failures)

Inherent EBNF representation limitations already documented in
`docs/CSS_EBNF_CONSTRAINTS.md`. Not bugs — fixing them requires restructuring the
grammar or moving the constraint to the semantic/render layer.

| Category | Failures | Explanation |
|---|---|---|
| `ebnf_constraint__vds_ordering` | 97 | VDS `\|\|` / `&&` allow any operand order; the EBNF translation fixes one canonical order, so CSS using a non-canonical value order fails |
| `ebnf_constraint__vendor_prefix_property` | 24 | Vendor-prefixed properties (`-webkit-*`, `-moz-*`, `-ms-*`) are intentionally omitted — documentation is inconsistent and they duplicate standard rules |
| `ebnf_constraint__url_prefix_ambiguity` | 1 | The optional `[ NamespacePrefixType ]` in `@namespace` greedily consumes `url` as an ident before the `url(…)` functional form can be tried |

## Undocumented constraints (128 failures)

Real EBNF/parser limitations not yet in `CSS_EBNF_CONSTRAINTS.md`.

| Category | Failures | Explanation |
|---|---|---|
| `declarationvaluetype_placeholder` | 95 | `DeclarationValueType` is defined as `string_literal` (only matches quoted strings); at-rules that use it as a free-form value bucket (`@media`, `@supports`, `@container`, `@property`, etc.) fail on any unquoted value |
| `keyframes_percent_selector` | 31 | `AtKeyframes` routes its block through `CssStyleSheet`, whose selector rules can't match percentage keyframe selectors (`0%`, `50%`, `100%`, `from`, `to`) |
| `viewport_unit_keyword_boundary` | 2 | `sv`, `lv`, `dv` are keyword terminals; gluon enforces a word boundary after each, so `svh`, `dvw`, `lvmin` etc. fail because the parser matches the prefix and rejects the suffix |

| Category | Failures | Explanation |
|---|---|---|
| `naming_bug__image_type_uses_url_keyword` | 88 | `ImageType` references the bare keyword terminal `url` (`"url"`) instead of the rule `UrlType`; the `(` after `url` is never consumed, so all `url("…")` image values fail |
| `naming_bug__trig_fn_not_in_calc_value` | 7 | `CosFn`, `SinFn`, `TanFn`, `AcosFn`, `AsinFn`, `AtanFn`, `Atan2Fn`, `ExpFn`, `LogFn`, `SqrtFn` are defined in `functions.ebnf` but never added to `MathFunctionType` or `CalcValueType` in `datatype.ebnf` |
| `naming_bug__font_face_stretch_vs_width` | 4 | `AtFontFaceWidthDesc` uses keyword `font-width` but real `@font-face` blocks use `font-stretch`; the descriptor never matches |
| `naming_bug__srgb_case` | 2 | `sRGB = "sRGB"` is case-sensitive; CSS uses lowercase `srgb` in `color(srgb …)`, so it never matches |
